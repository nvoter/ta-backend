# Веб-приложение для трудоустройства учебных ассистентов
## Описание
Клиент-серверное веб-приложение для организации процессов найма учебных ассистентов на дисциплины ФКН НИУ ВШЭ

Функциональные требования:
* Для студентов
  - Аутентификация по корпоративной электронной почте (с доменом edu.hse.ru)
  - Подача, просмотр и редактирование заявок на работу учебным ассистентом
  - Получение уведомлений о смене статусов заявок
* Для преподавателей и менеджеров
  - Аутентификация по корпоративной электронной почте (с доменом hse.ru)
  - Просмотр и изменение статусов заявок студентов
  - Управление сессиями (выполнение действий в системе от своего лица или от имени другого сотрудника)
* Для администраторов
  - Аутентификация по корпоративной электронной почте (с доменом hse.ru)
  - Управление кампаниями
  - Управление данными об образовательных программах и дисциплинах факультета (через загрузку xlsx-файла)
  - Управление информацией о студентах ФКН
## Архитектура
Серверная часть реализована на Java + Spring Boot с использованием микросервисной архитектуры. Каждый микросервис хранится в отдельном GitHub-репозитории:
* [users-microservice] (https://github.com/nvoter/ta-users-microservice) - аутентификация, управление сессиями, хранение информации о студентах и сотрудниках
* [disciplines-microservice] (https://github.com/nvoter/ta-disciplines-microservice) - управление данными об образовательных программах и дисциплинах ФКН
* [applications-microservice] (https://github.com/nvoter/ta-applications-microservice) - все действия, связанные с проведением кампаний: планирование и управление кампаниями, подача, обработка и изменение заявок на работу учебным ассистентом, сохранение и получение файлов зачетных книжек
* [notifications-microservice] (https://github.com/nvoter/ta-notifications-microservice) - уведомления и отправка писем на электронную почту
## Запуск
Для запуска необходимо в корне каждого микросервиса выполнить команду 
```
docker-compose up --build -d
```
Микросервисы можно запускать в произвольном порядке, но users-microservice и applications-microservice должны быть запущены после notifications-microservice

Перед тестированием подачи заявки от имени студента необходимо создать активную кампанию от лица администратора и бакет `gradebooks` в MinIO через [админ-панель](http://localhost:9001)

## Выполнение по критериям
### 1.1 Приложение поддерживает аутентификацию пользователей и контроль доступа к API
Аутентификация пользователей с выдачей JWT-токенов реализована в users-microservice. Для аутентификации необходимо воспользоваться **корпоративной электронной почтой**, на которую придет письмо с кодом подтверждения

Контроль доступа к API реализован посредством API-ключей

### 1.2 Приложение имеет gRPC или HTTP API (минимум четыре бизнес-метода: создание, получение, изменение, удаление)
Каждый микросервис имеет HTTP API. Все четыре типа эндпоинтов представлены в applications-microservice для управления кампаниями
